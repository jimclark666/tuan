## 功能概览

### Step1：对话日志标准化与评测切片
- 上传/粘贴对话日志 JSON
- 生成统一 Schema：turn 切片、场景识别、用户目标摘要、工具调用/检索信息、校验提示等
- 为后续评测提供可对齐输入

### Step2：LLM 智能评测（评分 + 稳定性 + AB 对比）
- 多维指标评分（示例：Intent Accuracy、Task Completion、Tool Use Correctness、Faithfulness、Helpfulness、Turn Efficiency 等）
- **稳定性评估**：同配置重复运行 N 次，输出均值/标准差/范围
- **AB 测试**：同一份日志对比 Model A vs Model B（两套 endpoint/config），输出差异与对比结果

### Step3：RCA 归因诊断（根因 + 证据 + 修复建议）
- 自动定位低分根因（P0/P1/P2 优先级）
- 证据轮次（turn_id）可跳转，支持追溯
- 输出可执行修复建议与预期收益（影响哪些指标）

### 策略生成与回归评测（闭环验证）
- 生成可落地策略文本（适合作为 system prompt / policy 参考）
- 一键“应用建议并回归评测”，展示优化前后分数变化与关键维度提升

## 快速开始（推荐操作路径）

> **强烈建议按 Step1 → Step2 → Step3 的顺序操作**。  
> 评测平台是“链式依赖”：后续步骤需要前置产物（标准化 turns、scoreResult 等）。

### 1）Step1：日志上传与标准化
1. 打开在线地址： https://aogalf.nocode.host/
2. 进入 **「日志上传」** Tab
3. 粘贴对话日志 JSON（或使用你准备的示例）
4. 点击 **「开始标准化处理」**
5. 在 **「标准化结果 / 校验结果」**确认已生成 turns 与摘要信息

### 2）Step2：LLM 智能评测（单模型 / AB）
1. 进入 **「LLM评测」** Tab
2. 配置模型（Provider / Base URL / Endpoint ID / API Key / Temperature）
3. 可选：
   - 开启 **稳定性评估**（N=1/3/5）
   - 切换 **AB 测试**（配置 Model A 与 Model B）
4. 点击 **「开始智能评测」**
5. 在 **「评分结果」**查看总分、维度详情、扣分原因与证据

### 3）Step3：RCA 归因诊断
1. 进入 **「RCA归因」** Tab
2. 点击 **「开始归因诊断」**
3. 查看 Top 根因、P0/P1/P2 问题、证据 turn 跳转、修复建议

### 4）策略生成 & 回归评测（闭环）
1. 在 RCA 页点击 **「生成可落地策略」**
2. 点击 **「应用建议并回归评测」**
3. 查看回归对比：总分变化、Top3 维度变化、是否解决根因等

## 模型配置说明（Volc Ark/火山方舟）

> 本项目支持外接大模型服务（示例为火山方舟）。

- **Base URL**：`https://ark.cn-beijing.volces.com/api/v3`
- **Endpoint ID**：通常形如 `ep-xxxx`（请在火山方舟“在线推理/推理接入点”页面复制）
- **API Key**：从对应平台创建（建议使用可随时作废的 Key）
- **Temperature**：建议 0.1–0.2（更稳定、也更利于结构化输出）

## 使用体验注意事项（必读）

### 1）外接大模型导致的等待时间（正常现象）
由于外接大模型，以下功能需要等待一段时间（取决于网络与模型负载）：
- **LLM 智能评测**
- **RCA 归因诊断**
- **生成可落地策略**
- **应用建议并回归评测**

建议演示策略：
- 优先使用较短对话样例
- 稳定性评估先用 **N=1**，跑通后再扩展到 N=3/5
- 若遇到超时/拥塞，稍后重试或降低输入长度（系统已做部分输入瘦身与调试可视化）

### 2）不要在“空状态”直接点击 RCA（否则可能出现空白页）
在刚打开页面、尚未完成 Step1/Step2 时，如果直接点击 **RCA 归因**按钮，可能出现页面空白或无数据状态。  
**正确方式**：从 Step1 开始按顺序执行（标准化 → 评分 → RCA），即可正常使用。  
若误触导致空白：刷新页面后从 Step1 重新开始。
